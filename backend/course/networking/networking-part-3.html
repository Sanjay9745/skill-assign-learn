üõ†Ô∏è Simple Network Programming Examples

Python Client (Sends Message):
import socket

# Create a socket
client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
client.connect(('server.com', 8080))

# Send message
client.send(b"Hello Server!")

# Receive response
response = client.recv(1024)
print(f"Server said: {response.decode()}")

client.close()
Python Server (Receives & Responds):
import socket

# Create server socket
server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.bind(('localhost', 8080))
server.listen(5)

while True:
  client, address = server.accept()
  message = client.recv(1024)
  print(f"Received: {message.decode()}")
  client.send(b"Hello Client!")
  client.close()

üîç What Happens When You Run This Code:

  1. Server Starts: Opens port 8080 and waits for connections
  2. Client Connects: Establishes TCP connection to server
  3. Three-Way Handshake: Client and server agree to communicate
  4. Data Exchange: Messages flow back and forth as raw bytes
  5. Connection Closes: Both sides agree to disconnect
üìû Socket Programming Analogy: Making a Phone Call

TCP Socket = Phone Call: You dial, wait for answer, have conversation, hang up together
UDP Socket = Walkie-Talkie: Just broadcast and hope someone hears you
Socket Address = Phone Number: IP address + port number tells you exactly who to call
Port = Extension Number: Different services on same computer use different ports

üåü Real-World API Example: Weather App

// JavaScript - Fetching weather data
async function getWeather(city) {
  // API call to OpenWeatherMap
  const response = await fetch(
    `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=YOUR_KEY`
  );

  const data = await response.json();

  // Extract useful information
  return {
    temperature: data.main.temp,
    description: data.weather[0].description,
    humidity: data.main.humidity
  };
}

// Usage
getWeather("Tokyo").then(weather => {
  console.log(`It's ${weather.temperature}¬∞C and ${weather.description}`);
});

üèóÔ∏è Cloud Architecture Patterns

üì± Serverless

Write Functions, Not Servers

  • Code runs only when needed
  • Auto-scales from 0 to millions
  • Pay per execution
  • Perfect for APIs and webhooks
// AWS Lambda function
exports.handler = async (event) => {
  return {
    statusCode: 200,
    body: 'Hello from the cloud!'
  };
};
üê≥ Containers

Package Everything Together

  • App + dependencies in one package
  • Runs identically everywhere
  • Easy to scale and manage
  • Docker + Kubernetes power
# Dockerfile
FROM node:18
COPY . .
RUN npm install
CMD ["npm", "start"]

# Deploy to cloud
docker build -t my-app .
docker push registry/my-app
üåê CDN

Global Content Delivery

  • Copies of your content worldwide
  • Users get data from nearest location
  • Faster loading, lower latency
  • Handles traffic spikes automatically
üîÑ Auto-Scaling

Grows and Shrinks Automatically

  • Monitors your app's load
  • Adds servers when busy
  • Removes servers when quiet
  • Saves money and ensures performance
