What is CSS? ü§î

CSS stands for Cascading Style Sheets. It's a stylesheet language used to describe the presentation of a document written in HTML. CSS dictates how HTML elements should be rendered on screen, on paper, or in other media.

CSS Syntax: How to Write CSS Rules üìú

A CSS rule consists of a selector and a declaration block.

  • The selector points to the HTML element you want to style.
  • The declaration block contains one or more declarations separated by semicolons.
  • Each declaration includes a CSS property name and a value, separated by a colon.

/* This is a CSS comment */
h1 {
  color: blue;
  font-size: 24px;
  text-align: center;
}

p {
  color: green;
  font-family: Arial, sans-serif;
}

Applying CSS to HTML: 3 Main Ways üîó

You can add CSS to your HTML documents in three ways:

1. External CSS (Most Recommended)

Styles are defined in a separate .css file (e.g., style.css). You link this file in the HTML <head> section.


<!DOCTYPE html>
<html>
<head>
  <title>My Page</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  ...
</body>
</html>
                

/* styles.css */
body {
  background-color: lightblue;
}
h1 {
  color: navy;
}

Pros: Clean separation, reusable across multiple pages, easier maintenance.

2. Internal CSS (Embedded)

Styles are placed within a <style> tag in the HTML <head> section. Useful for single-page styling or specific page overrides.


<!DOCTYPE html>
<html>
<head>
  <title>My Page</title>
  <style>
    body {
      background-color: lightyellow;
    }
    p {
      color: maroon;
    }
  </style>
</head>
<body>
  ...
</body>
</html>

Pros: Single file, useful for quick tests or email templates.
Cons: Not reusable for other pages, mixes structure and presentation.

3. Inline CSS (Least Recommended)

Styles are applied directly to HTML elements using the style attribute. Generally avoided for maintainability reasons.


<!DOCTYPE html>
<html>
<head>
  <title>My Page</title>
</head>
<body>
  <h1 style="color: red; text-align: center;">
    This is a heading
  </h1>
  <p style="font-size: 18px;">
    This is a paragraph.
  </p>
</body>
</html>

Pros: Quick for very specific, one-time styles.
Cons: High specificity, hard to manage, mixes content and presentation significantly.

üëî Analogy: Styling Instructions

‚Ä¢ Internal CSS: Like specific styling notes written at the beginning of one particular clothing pattern (HTML page).
‚Ä¢ Inline CSS: Like sewing a special button (style) directly onto one part of a garment (HTML element).

CSS Selectors: Targeting Your Elements üéØ

Selectors are patterns that match HTML elements. Here are some fundamental types:

Element Selector

Selects all elements of a specific type.


p { /* Selects all 
 elements */
  color: #333;
}
div { /* Selects all 
 elements */
  margin-bottom: 10px;
}

Class Selector

Selects all elements with a specific class attribute. Uses a period (.) prefix.


.highlight { /* Selects  */
  background-color: yellow;
}
.button-primary {
  padding: 10px 15px;
}
                    

<p class="highlight">Important text.</p>
<button class="button-primary">Click Me</button>

ID Selector

Selects a single element with a specific ID attribute. Uses a hash (#) prefix. IDs must be unique per page.


#main-header { /* Selects  */
  font-size: 32px;
}
#sidebar {
  width: 200px;
}
                    

<header id="main-header">...</header>
<aside id="sidebar">...</aside>

Attribute Selector

Selects elements based on an attribute or attribute value.


input[type="text"] {
  border: 1px solid #ccc;
}
a[target="_blank"] { /* Links opening in new tab */
  color: green;
}

Grouping Selectors

Apply the same styles to multiple selectors by separating them with commas.


h1, h2, h3 {
  font-family: Georgia, serif;
  color: #2a2a2a;
}

Descendant Combinator (space)

Selects elements that are descendants of a specified element.


nav ul li { /* Selects 
  •  inside 
       inside 
       */
  list-style-type: none;
}

    The CSS Box Model: Every Element is a Box üì¶

    In CSS, every HTML element is treated as a rectangular box. The CSS Box Model describes how these boxes are structured and includes properties for content, padding, border, and margin.

    • Content: The actual content of the box, where text and images appear. Sized by width and height.
    • Padding: Clears an area around the content. Padding is transparent.
    • Border: A border that goes around the padding and content.
    • Margin: Clears an area outside the border. The margin is transparent and separates the box from other elements.
    
.my-box {
  width: 300px;
  height: 150px;
  padding: 20px; /* 20px padding on all sides */
  border: 5px solid red;
  margin: 30px auto; /* 30px top/bottom, auto left/right for centering */
  background-color: lightgoldenrodyellow;
}
    üñºÔ∏è Analogy: Picture Frame

    ‚Ä¢ Content: The photograph itself.
    ‚Ä¢ Padding: The matting around the photograph, inside the frame.
    ‚Ä¢ Border: The picture frame.
    ‚Ä¢ Margin: The space between this picture frame and other frames on the wall.

    Common CSS Properties to Get Started ‚ú®

    Here are some frequently used CSS properties to control the appearance of your elements:

    Text Styling

    color: Sets text color.
    p { color: #336699; }
    font-family: Specifies font.
    body { font-family: Arial, sans-serif; }
    font-size: Sets text size.
    h1 { font-size: 2.5em; }
    font-weight: Sets font boldness.
    strong { font-weight: bold; }
    text-align: Aligns text.
    .center { text-align: center; }
    line-height: Sets space between lines.
    p { line-height: 1.8; }
    text-decoration: Adds lines (underline, overline, etc.).
    a { text-decoration: none; }

    Backgrounds & Borders

    background-color: Sets background color.
    div { background-color: #f0f0f0; }
    background-image: Sets background image.
    body { background-image: url('bg.png'); }
    border: Shorthand for border properties.
    .box { border: 1px solid black; } (width, style, color)
    border-radius: Creates rounded corners.
    button { border-radius: 5px; }

    Sizing & Spacing (Box Model)

    width / height: Sets element dimensions.
    img { width: 100px; height: auto; }
    padding: Inner space.
    .card { padding: 15px; }
    margin: Outer space.
    p { margin-bottom: 20px; }

    Display & Layout (Basic)

    display: Controls element's display type.
    li { display: inline-block; }
    (Common values: block, inline, inline-block, none, flex, grid)
    position: Sets positioning scheme.
    .modal { position: fixed; }
    (Values: static, relative, absolute, fixed, sticky)

    Specificity and The Cascade: Who Wins? ü•á

    "Cascading" in CSS refers to how styles are applied when multiple rules target the same element. The browser follows a set of rules to determine which style takes precedence:

    1. Origin and Importance: User agent stylesheets (browser defaults) < Author stylesheets (your CSS) < User stylesheets (user-defined overrides). Styles marked with !important get higher priority within their origin. (Use !important sparingly!)
    2. Specificity: More specific selectors override less specific ones.
      • Inline styles (in HTML style attribute) - Highest specificity
      • IDs (#example)
      • Classes (.example), attribute selectors ([type="text"]), pseudo-classes (:hover)
      • Elements (div), pseudo-elements (::before) - Lowest specificity
    3. Source Order: If two selectors have the same specificity, the one that appears later in the CSS or in the HTML (for inline styles) wins.
    
p { color: blue; } /* Element selector - low specificity */
.my-paragraph { color: green; } /* Class selector - medium */
#intro { color: red; } /* ID selector - high */

/* If an element is 
    , it will be RED. */
/* If it's 
    , it will be GREEN. */
/* If it's just 
    , it will be BLUE. */

h2 { color: purple !important; } /* This will likely override other h2 color rules */
h2 { color: orange; } /* This orange will be ignored if the !important rule exists */
    üì¢ Analogy: Giving Orders

    Imagine you're decorating a room (HTML element):
    ‚Ä¢ Element selector (p { color: blue; }): A general instruction: "All walls should be blue."
    ‚Ä¢ Class selector (.accent-wall { color: red; }): A more specific instruction: "Walls with the 'accent' label should be red." (Overrides general instruction for those walls).
    ‚Ä¢ ID selector (#feature-wall { color: gold; }): A very specific instruction for one unique wall: "The 'feature' wall must be gold." (Overrides class and general).
    ‚Ä¢ Inline style (style="color: silver;"): Someone painting a silver swatch directly on the 'feature' wall. (Highest specificity, overrides ID).
    ‚Ä¢ !important: The building owner shouting "ALL WALLS MUST BE WHITE, NO EXCEPTIONS!" (Overrides almost everything).

    Let's Style Your First Page! üöÄ

    Remember the "My First HTML Page" from Part 1? Let's add some style to it. First, create a new file named mystyle.css in the same folder as your index.html.

    1. Link your CSS file in index.html:

    
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First HTML Page</title>
    <link rel="stylesheet" href="mystyle.css"> <!-- Add this line -->
</head>
<body>
    <header>
        <h1>Welcome to My Website</h1>
    </header>
    
    <nav>
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </nav>
    
    <main>
        <section class="about-me">
            <h2>About Me</h2>
            <p>Hello! I'm learning HTML and CSS, and this is my first styled webpage.</p>
            <p>I'm excited to learn web development!</p>
        </section>
        
        <section class="my-hobbies">
            <h2>My Hobbies</h2>
            <ul>
                <li>Coding</li>
                <li>Reading</li>
                <li>Photography</li>
            </ul>
        </section>
    </main>
    
    <footer>
        <p>© 2023 My First Website. All rights reserved.</p>
    </footer>
</body>
</html>

    (Notice I added class="about-me" and class="my-hobbies" to the sections for easier targeting with CSS).

    2. Add styles to mystyle.css:
