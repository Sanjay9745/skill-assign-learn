Conditional Statements: Making Choices!

Sometimes, your code needs to make decisions. That's where conditional statements like if, else if, and else come in handy! They let your program do different things based on whether something is true or false.

The Simple if Statement

An if statement runs a block of code only if a certain condition is true.

Analogy Time! üö¶ Imagine you're at a traffic light. If the light is green (condition is true), you go (action). If it's not green, you don't do that specific "go" action. 

let temperature = 30; // degrees Celsius

if (temperature > 25) {
  console.log("It's a hot day! Wear shorts! ‚òÄÔ∏è");
}
// Output: It's a hot day! Wear shorts! ‚òÄÔ∏è

Visualizing an if Statement: The Two Roads

The if...else Statement

What if you want to do something else if the condition is false? That's what else is for!

Analogy Time! üç¶ You want ice cream. If your favorite flavor (e.g., chocolate) is available, you get that. Else (if it's not available), you get vanilla. 

let age = 15;

if (age >= 18) {
  console.log("You are old enough to vote! üó≥Ô∏è");
} else {
  console.log("Sorry, you are too young to vote. üòï");
}
// Output: Sorry, you are too young to vote. üòï

The if...else if...else Ladder

For checking multiple conditions in a sequence, you can use an else if ladder. It's like a series of questions.

Analogy Time! üéÆ Choosing a game:
  • If you feel like building: Play Minecraft.
  • Else if you feel like an adventure: Play Zelda.
  • Else if you want to relax: Play Stardew Valley.
  • Else (if none of the above): Read a book.

let score = 85;
let grade;

if (score >= 90) {
  grade = "A üéâ";
} else if (score >= 80) {
  grade = "B üëç";
} else if (score >= 70) {
  grade = "C üòä";
} else if (score >= 60) {
  grade = "D ü§î";
} else {
  grade = "F üò•";
}
console.log("Your grade is: " + grade); // Output: Your grade is: B üëç

Comparison and Logical Operators

To make conditions, we use comparison operators (like >, <, === for strict equality) and logical operators (&& for AND, || for OR, ! for NOT).

let isWeekend = false;

if (hour < 9 || hour > 17 || isWeekend) {
  console.log("The office is closed. üò¥");

  console.log("The office is open! üíº");

Loops: Repeating Actions!

Loops are super useful when you want to do something over and over again without writing the same code multiple times.

The for Loop

A for loop is great when you know how many times you want to repeat an action.

It has three parts, separated by semicolons ( ; ):

  1. Initialization: Code that runs once at the very beginning (e.g., let i = 0). Sets up your loop variable.
  2. Condition: Checked before each loop. If it's true, the loop runs. If false, the loop stops (e.g., i < 5).
  3. Final Expression (Update): Code that runs after each loop iteration (e.g., i++ to count up).
Analogy Time! ü§∏ Doing 5 jumping jacks:
  1. Initialization: Say "Okay, I'll start counting my jumping jacks from 1." (let count = 1)
  2. Condition: "Have I done less than or equal to 5 jumping jacks?" (count <= 5)
  3. Code Block: If yes, do one jumping jack!
  4. Final Expression: Add one to my count. (count++)
  5. Repeat from step 2.

// Print numbers from 1 to 5
for (let i = 1; i <= 5; i++) {
  console.log("Number: " + i + " üéà");
}
// Output:
// Number: 1 üéà
// Number: 2 üéà
// Number: 3 üéà
// Number: 4 üéà
// Number: 5 üéà

// Looping through an array
let friends = ["Alex", "Sam", "Jordan"];
for (let i = 0; i < friends.length; i++) {
  console.log("Hello, " + friends[i] + "! üëã");
}
// Output:
// Hello, Alex! üëã
// Hello, Sam! üëã
// Hello, Jordan! üëã

Visualizing a for Loop

The while Loop

A while loop keeps repeating as long as its condition is true. You set up the starting point before the loop, check the condition at the start of each repetition, and you must change something inside the loop to eventually make the condition false (otherwise, you get an infinite loop! üò±).

Analogy Time! üçø Eating popcorn:
  1. Initialization: You have a full bowl of popcorn. (let popcornLeft = true;)
  2. Condition: "Is there still popcorn left?" (while (popcornLeft))
  3. Code Block: If yes, eat some popcorn! Then, check if the bowl is empty. If it is, set popcornLeft = false;.
  4. Repeat from step 2.

let countdown = 3;

while (countdown > 0) {
  console.log(countdown + "... üöÄ");
  countdown--; // IMPORTANT: Change the variable so the loop eventually stops!
}
console.log("Blast off!");
// Output:
// 3... üöÄ
// 2... üöÄ
// 1... üöÄ
// Blast off!

for vs while

  • Use a for loop when you generally know how many times you want to loop (e.g., for each item in an array, or 10 times).
  • Use a while loop when you want to loop as long as a condition is true, and you don't necessarily know upfront how many iterations that will be.

Functions: Reusable Code Spells!

Functions are like named blocks of code that you can run whenever you need them. They help you organize your code and avoid repeating yourself (this is called the DRY principle: Don't Repeat Yourself!).

Defining and Calling Functions

First, you define (create) a function, then you call (run) it.

Analogy Time! üìú
  • Defining a function is like writing down a recipe for your favorite cookies. The recipe has a name (function name) and steps to follow (code inside).
  • Calling a function is like actually taking out the recipe and baking those cookies! You can bake them many times using the same recipe.

// 1. Defining the function (writing the recipe)
function sayHello() {
  console.log("Hello there, awesome coder! üëã");
}

// 2. Calling the function (using the recipe)
sayHello(); // Output: Hello there, awesome coder! üëã
sayHello(); // You can call it again! Output: Hello there, awesome coder! üëã

Parameters and Arguments: Giving Info to Functions

Functions can take parameters (placeholders for information they need) and you give them arguments (the actual information) when you call them.

Analogy Time! üíå Imagine a function that writes a greeting card:
  • Parameter: The recipe for the card might say "Write 'Dear [name]'". Here, `name` is a parameter.
  • Argument: When you actually write a card, you fill in the blank: "Dear [Alex]". Here, "Alex" is the argument.

// Function with one parameter: 'name'
function greetUser(name) {
  console.log("Welcome to the game, " + name + "! üèÜ");
}

greetUser("PixelPioneer"); // "PixelPioneer" is the argument
// Output: Welcome to the game, PixelPioneer! üèÜ

greetUser("CodeWizard");   // "CodeWizard" is the argument
// Output: Welcome to the game, CodeWizard! üèÜ

// Function with two parameters: 'num1' and 'num2'
function addNumbers(num1, num2) {
  let sum = num1 + num2;
  console.log("The sum is: " + sum);
}

addNumbers(5, 3); // 5 and 3 are arguments. Output: The sum is: 8
addNumbers(10, 20); // Output: The sum is: 30

Return Values: Getting Info Back from Functions

Functions can also give back a result using the return keyword. When a function hits a return statement, it stops running and sends the value back.

Analogy Time! ‚ûï Think of a calculator function. You give it two numbers (arguments), it does the calculation, and then it returns the answer to you. 

function calculateArea(width, height) {
  let area = width * height;
  return area; // Sends the calculated area back
}

let roomArea = calculateArea(10, 5); // The function call becomes the returned value
console.log("The room area is: " + roomArea + " square meters."); // Output: The room area is: 50 square meters.

let anotherArea = calculateArea(7, 3);
console.log("Another area: " + anotherArea); // Output: Another area: 21

If a function doesn't have a return statement, or just has return; with no value, it automatically returns undefined.
